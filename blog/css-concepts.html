CSS Concepts

09/05/2015

Of the topics covered during this last week of unit 1, phase-0, I know that I hadn't exactly mastered the display property. Specifically, what is the difference between the inline and inline-block display values. So naturally, I opened up my browser to GOOGLE, and began to click away.

After merely a few clicks, I scrolled down a page of stack overflow and discovered a remarkably concise list of the differences. But of course, it wouldn't do you any good for me to just paste that and not supply any context. So here goes:

When you designate elements of a parent element to display "block" you are saying that you want the elements to sit on top of one another, presumably like blocks. That is to say, the images will stack. Because there is one element per line, you have free reign over margins, borders, and padding.

And when you designate elements of a parent element to display "inline" you are saying that you want the elements to line up next to each other. Because this is the display you want, css limits what you can do with them, in order to honor the goal of this display value. So inline elements can not set width and height, and margins cannot be added to the top or bottom.

Finally, when you designate elements of a parent element to display "inline-block" you are saying that you want the elements to behave like block elements, except you don't necessarily want a page break after each element.
