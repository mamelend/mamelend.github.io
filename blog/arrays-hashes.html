Arrays and Hashes

09/13/2015

Welcome back, Reader. I'm glad you've come back to join me for the end of week four. This week was all about Ruby. More specifically, we were introduced to some of its basic classes, variables, methods, conditionals, flow control, rspec and error finding, and finally iterators and collections.

I wanted to take a few minutes and just layout my understanding of collections, of course for your benefit, but also in hopes that it will reinforce my own understanding of the various concepts. In Ruby, we've been introduced to two collections. One is called the Array, and the other is called the Hash.

Arrays are indexed collections that are accessed through referencing their index which is an integer. An array may look something like the following:

new_array = [ “apple”, “plum”, “nectarine”, “grapefruit” ]
new _array[0] => "apple"
new_array[3] => "grapefruit"

Arrays live inside these [] brackets. The convention, when creating a new array, is to set a variable name = to a given list of items, within square brackets and separated by commas. This is called a literal declaration. As in, you are literally declaring all of the contents of the array. You can also use the constructor Array.new to make a new empty array.

new_array = Array.new

Arrays are useful for lists that need to be accessed quickly, and that already have some sort of uniformity. If you have a list that isn't in any easily defined order, it's better to use a Hash.

Hashes, are almost identical to Arrays. They are similar in that each item of an array or hash has both a value and an index, but they are different in that indexes for hashes are refered to as keys and they can be anything. Oh! And hashes use braces rather than square brackets.

new_hash = { 'item' => 'box',
'animal' => 'fox',
'food' => 'lox',
}
new_hash = Hash.New

Again, hashes are useful if there isn't any sort of consecutive number structure that can be used to id each value.
