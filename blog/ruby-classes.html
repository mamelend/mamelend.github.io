A little about Ruby classes

09/27/2015

Classes are the blue prints used to create objects in a running system. The following is a very basic class intended for instructional purposes. No cats were harmed during the creation of this blog post:

This class basically allows us to hold the name and color of a particular cat (or instance of the Class Cat), and allow us to print out a meow message and another message that includes the above information. Storing information in instance variables with the "@" sign allows for us to use that information anywhere else in the class. For example. as I just mentioned, the @name and @color variables are used again in the display method, in order to allow us to print out that information about the cat.

As you can see, there was no need to provide the name and color arguements again. Classes are very powerful for compartmentalizing data of bigger systems. Think of it this way, what if I wanted to write a program that accessed all of the different pets in a household? Rather than handle all of the different factors in one Pets class, I can divide them up into maybe Cat, Dog, Fish, and Bird classes. Each could have its own distinct variables and features that might be more appropriate or specific for the given pet. For example, I could create a Dog class that includes a variable for the breed.
