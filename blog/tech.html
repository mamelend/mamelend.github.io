The wonders of Recursion!

10/11/2015

Recursion, in computer science, decribes the process by which a function (or method) is defined, in part, by a call to itself. Recursive functions have two cases. A recursive case where the method calls itself, and a base case where the method does not call itself. The recursive case performs a subtask, or set of subtasks, until it satisfies the requirement for the base case, which ends the successive calls and begins to resolve the nest of calls in order to return a solution to the original problem. Take a look at a classic example, finding the factorial of a number.

Passing in a 3 would initiate the following: On the first pass through the intial if statement, 3 noes not equal 0 so it would hit the else. The else would call the factorial method, except it would decrement the argument by 1. This would continue to occur until ultimately number = 0, and so that call would return 1 and this would begin by resolving the previous call, and continue until all calls are resolved.
