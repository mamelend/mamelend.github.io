Ruby Hashes vs. JavaScript Objects

10/04/2015

This week was all about picking up Javascript. I don't think I would be doing anyone a favor if I failed to mention how limiting and redundant Javascript felt compared to Ruby. But there was one area in particular, while learning the fundamentals of Javascript, that sort of surprised me in how wonderful it was to use; that would be Javascript Objects versus Ruby Hashes.

These both look syntactically similar. Hashes and Objects use the colon in literal definitions and calls (see below) and they function pretty much the same in terms of being structured as key and value pairs. They can also both hold arrays, integers, strings, other objects, or a combination of any or all of these.

How do they differ? A JavaScript Object is a collection of properties that references other objects, whereas a Ruby Hash contains instance variables which refer to other objects. In other words, Ruby Hashes hold variables which point to other objects elsewhere in memory, while JavaScript Objects hold properties which are tied specifically to that object.
