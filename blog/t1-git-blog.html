Bourne Again

08/20/2015

Well, I’m just about wrapping up my assignments for the first week of Dev Bootcamp and I have to admit that I didn’t expect to learn so much in just the first few days. We haven’t touched much code yet, but that’s okay because what we have looked at has been super interesting. This first week has introduced us to the terminal, or the command line; the shell, or bash specifically; and Version Control, Git, and GitHub, which I’ll go a little bit further into below:

Version Control is a system that stores changes to a given set of data (source code, editorial prose, etc.) over time so that users can keep track of what changes have been made to the data, and to recall specific states of that data. Version Control has many benefits, but principal among them are that they allow multiple users to collaborate on specific sets of data, adding and editing the same set of data, and merging the changes all together, in real time. Also, Version Control provides for users to backup the data at specific points throughout its lifecycle and then recall the data, sort of the way you might restore your operating system from a backup, or the way you might load up a video game from a specific checkpoint (not just the last checkpoint). Lastly, I want to empahsize how incredibly useful it must be for users to have a complete record of changes, and documentation on why those changes were made, in order that they might come back to a certain set of data, say years later, and understand what happened.

Git tracks changes using a two-way encryption mechanism that basically takes snapshots of states of data, and then compares it with previous snapshots, in order to efficiently store the different states of your data as you work through a project. For example, if you have 10 files in a particular set of data, but only make changes to one, those changes will be tracked along with how it relates to the rest of the data.

GitHub, is a free online Git repository hosting service, that allows users to save and collaborate on different projects and also stores information on how the projects can be accessed and (legally) used. As a collaborative tool, GitHub is useful for opening up your code to a given group of people to take a look at and recommend small specific changes. Github also has a social network component, wikis, and various task management tools.

I know that I’m just scratching the surface here when it comes to most of these: Bash, Git, and GitHub. But I feel incredibly motivated in continuing to learn how to use these tools and services, and I can already start to see the world of difference it will make when approaching code and how I can bridge it over to creating something more tangible and useful for others.
