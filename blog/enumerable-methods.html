Stop writing FOR loops...sometimes.

09/20/2015

Lets talk a little bit about enumerable methods today! Specifically, I wanted to take a look at map, which is based on each, and is useful for iterating through an array, in the way you might use a FOR loop. But there is a better way!

So, just like each, map iterates through each element of a collection class. The key difference is that map has the ability to transform each element of that collection, where as each can not on its own. So check out the following example:

Here we have an array named col, with the contents of [1, 2, 3]. On the next line we invoke or call the map method which transforms the elements of col, and then saves it to transformed_col. We print out both and see that it yields the desired effect!

The original col method still yields the original contents because the original array was never transformed. But as you will see below, the destructive version of map, or map!, does the trick just fine!

map is super useful for either changing the contents of the original array temporarily, in order to manipulate or run other operations, OR for transforming the original array althogether.

Lastly, I want to quickly point out the each will not transform the array, and also doesn't have a destructive version. Happy Coding!
