Ruby Enumerables Cheat Sheet

Disclaimer: All of the information contained here and the examples were highlighted during my first reading through The Well-Grounded Rubyist by David A. Black. I've typed it up and presented it in a super concise format in order to easily refer back to these great examples.

#each

This is what I do: Iterates through each element in an enumerable.

This is what I look like:

WGR: “Exactly what each does will vary from one class to another. In the case of an array, each yields the first element, then the second, and so forth. In the case of a hash, it yields key/value pairs in the form of two-element arrays. In the case of a file handle, it yields one line of the file at a time. Ranges iterate by first deciding whether iterating is possible (which it isn’t, for example, if the start point is a float) and then pretending to be an array.”

#detect or #find

This is what I do: Returns the first element in the enumerable object for which the code block provided returns true.

This is what I look like:

WGR: “find…returns the first element in an enumerable object for which the code block provided returns true. find works by calling each. each yields items, and find uses the code block we’ve given it to test those items one at a time for a match.”

#find_all or #select

This is what I do: Returns a new collection containing all the elements of the original collection that match the criteria in the code block.

This is what I look like:

WGR: find_all (the same method as select) returns a new collection containing all the elements of the original collection that match the criteria in the code block, not just the first such element (as with find). If no matching elements are found, find_all returns an empty collection object.

#drop

This is what I do: Returns an array of all of the elements outside the first __ number of elements where __ is the number passed in as an argument.

#take_while

This is what I do: Iterates through each element to identify which arguments return true for the corresponding code block then returns an array of the first __ number of elements where __ is the number passed in as an argument.

This is what I look like:

>> states = %w{ NJ NY CT MA VT FL } => ["NJ", "NY", "CT", "MA", "VT", "FL"] >> states.take_while {|s| /N/.match(s) } => ["NJ", "NY"]

#drop_while

This is what I do: Iterates through each element to identify which arguments return true for the corresponding code block then returns an array of all of the elements outside the first __ number of elements where __ is the number passed in as an argument.

#each_slice and #each_cons

This is what I do: each_slice handles each element only once, whereas each_cons takes a new grouping at each element and thus produces overlapping yielded arrays.

#inject or # reduce

This is what I do: Initializes an accumulator object and then iterates through an enumerable object, performing a calculation on each iteration and resetting the accumulator, for purposes of the next iteration, to the result of that calculation. If you don’t supply an argument to inject, it uses the first element in the enumerable object as the initial value for acc.

This is what I look like:

#map or #collect

This is what I do: Returns an array that is the same size as the original enumerable. Its elements consist of the accumulated result of calling the code block on each element in the original object in turn.
